# Quick Start Workflow

For when you want to skip the full pipeline and just start building.
Still follows TDD — tests before code.

1. Ask the user what they want to build (2-3 questions max)
2. Write a brief spec directly to `docs/specs/00-quick-spec.md`
3. Write tests based on the spec
4. Implement code to make the tests pass
5. Skip containerization and review unless requested

Use this when the user says things like "just build it", "skip the interview",
or "I already know what I want".
