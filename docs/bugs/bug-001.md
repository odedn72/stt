# Bug 001: LevelMeter SILENT threshold deviates from spec

**File & line:** `src/systemstt/audio/level_meter.py`, line 93

**Spec reference:** `docs/specs/02-audio-capture.md`, section 6, note 6

**Expected behavior:**
Per the spec, the SILENT threshold should be `rms_db < -60`:

> - SILENT: rms_db < -60
> - TOO_QUIET: -60 <= rms_db < -40

The `_classify` method should use `-60.0` as the SILENT/TOO_QUIET boundary.

**Actual behavior:**
The `_classify` method uses `-65.0` as the SILENT threshold:

```python
if rms_db < -65.0:
    return AudioLevel.SILENT
```

This means audio with RMS between -65 dBFS and -60 dBFS is incorrectly classified as TOO_QUIET instead of SILENT. The docstring on `_classify` even acknowledges this deviation: "The SILENT threshold is set at -65 dBFS to properly categorize very quiet signals (amplitude ~0.001) as TOO_QUIET rather than SILENT." However, the root cause was a test fixture issue (amplitude 0.001 producing -63 dBFS), not a spec issue. The fixture has been corrected; the threshold should be restored to match the spec.

**Fix:**
Change line 93 from:
```python
if rms_db < -65.0:
```
to:
```python
if rms_db < -60.0:
```

And update the `_classify` docstring to remove the deviation note.

**Reproduction:** Create a test with a sine wave at amplitude ~0.0008 (RMS ~ -64 dBFS). Per spec it should be SILENT, but the current implementation classifies it as TOO_QUIET.
